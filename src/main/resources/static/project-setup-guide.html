What Is This

このアプリは何をするものか

このプロジェクトは、Spring Bootで作られたタスク管理APIです。 タスクの作成、一覧取得、1件取得、更新、削除をHTTP経由で行えます。

役割

タスク管理用のREST APIを提供するアプリです。

サーバー側

Spring Boot がAPI本体として動きます。

データ保存先

PostgreSQL にタスク情報を保存します。

動かし方

Docker Compose でAPIとDBをまとめて起動できます。

File And Tool Relation

どのファイルが、どのツールに関係しているか

この対応がわかると、修正したい内容に応じて見る場所を選びやすくなります。

VSCode で主に触るもの

  • src/main/java/...: Javaコード本体
  • src/main/resources/...: 設定や静的ページ
  • README.md: 使い方の説明

Docker に関係するもの

  • Dockerfile: APIコンテナの作り方
  • docker-compose.yml: app と db をまとめる設定
  • .env: Docker Compose の環境変数

Spring Boot / Gradle に関係するもの

  • build.gradle: 依存関係とビルド設定
  • settings.gradle: プロジェクト名
  • application.properties: DB接続やJPA設定

Git / GitHub に関係するもの

  • .gitignore: Gitに含めないファイル
  • .gitattributes: 改行やバイナリ扱いの設定
  • GitHub: リモート保存先

Glossary

初学者向けの用語集

初めて学ぶ時につまずきやすい言葉を、できるだけ短く整理しています。 まずはここをざっと見てからコードやDockerの説明に戻ると理解しやすいです。

API

他のアプリから使える機能の入口です。このプロジェクトでは /tasks などのURLがAPIです。

Spring Boot

JavaでWebアプリやAPIを作りやすくするフレームワークです。

Framework

アプリを作るための土台です。Spring Boot は Java の代表的なフレームワークです。

JDK

Javaのプログラムを作って実行するための開発キットです。Java 17 を使っています。

Gradle

ライブラリ管理、ビルド、テスト実行をまとめて行うツールです。

Dependency

自分で全部作らず、外部ライブラリとして取り込んで使う部品のことです。

Controller

HTTPリクエストを受け取る入口です。どの処理を呼ぶかを決めます。

Service

業務ロジックを書く場所です。作成日時設定や存在確認などを担当します。

Repository

DBへ保存・取得する窓口です。SQLに近い役割をJava側で扱います。

Entity

DBテーブル1件分のデータをJavaクラスで表したものです。このプロジェクトでは Task が該当します。

DTO

受け取り用、返却用など、データの受け渡しだけに使うクラスです。

JPA

JavaオブジェクトとDBテーブルを対応づけて扱いやすくする仕組みです。

PostgreSQL

データを保存するデータベースです。タスク情報はここに入ります。

Docker

アプリを「コンテナ」という実行単位で動かす仕組みです。PC環境差を減らせます。

Container

アプリを動かすための小さな実行環境です。今回は app と db の2つがあります。

Docker Compose

複数コンテナをまとめて起動・停止するための設定とコマンドです。

Image

コンテナを作る元になる設計図です。Dockerfile から作られます。

Port

アプリが外部と通信するための番号です。APIは 8080、DBは 5432 を使っています。

.env

ユーザー名やパスワードなど、環境ごとに変わる設定を書くファイルです。

Git / GitHub

Gitは変更履歴を残す仕組み、GitHubはその履歴をネット上で保管・共有する場所です。

From Scratch

このプロジェクトを1から作るなら、どんな順番になるか

ここでは「まったく何もない状態」から、この構成にたどり着くまでの流れを並べています。

01

開発ツールを用意する

VSCode、Java 17、Docker、Git をインストールします。

02

Spring Boot のひな形を作る

Spring Initializr などで、Web / JPA / PostgreSQL / Validation の依存を含むプロジェクトを作成します。

03

DBの設計を決める

tasks テーブルに何を保存するか決めて、対応するEntityクラス Task.java を作ります。

04

Repository を作る

DBに保存・検索するための窓口として TaskRepository を作ります。

05

Service を作る

作成日時更新や存在確認など、業務処理を TaskService にまとめます。

06

Controller を作る

/tasks 系のエンドポイントを定義し、HTTPリクエストを受け取れるようにします。

07

DTO と例外処理を追加する

入力用と出力用のDTOを作り、バリデーションや404処理を整えます。

08

Docker 化する

Dockerfile と docker-compose.yml を作り、APIとDBをまとめて起動できるようにします。

09

Swagger や学習ページを追加する

自分で触って確認しやすいようにAPIドキュメントや学習用ページを足します。

10

GitHub で管理する

Gitで履歴を作り、GitHubにpushしてバックアップと共有ができるようにします。

Checklist

このページを見ながら再現するためのチェックリスト

1から同じものを作る時は、下の順で確認すると進めやすいです。

準備

  • Java 17 を入れたか
  • Docker が動くか
  • VSCode でプロジェクトを開けるか
  • Git が使えるか

アプリ

  • Controller / Service / Repository を作ったか
  • Entity と DTO を作ったか
  • 例外処理を入れたか
  • application.properties を設定したか

実行

  • docker compose up --build で起動できるか
  • http://localhost:8080 に接続できるか
  • Swagger UI が開けるか
  • タスク作成APIが動くか
